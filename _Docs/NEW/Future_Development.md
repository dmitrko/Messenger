# Дорожная Карта Развития NostaChat (NEW)

## Планируемые функции

### 1. Звуковая Среда (Audio Environments)
- **Цель**: Вернуть оригинальные звуки ICQ («Uh-oh!», звук печатания, звук отправки сообщения, Buzz).
- **Интеграция**: Использование библиотеки `howler.js` для управления аудио-каналами. Звук гудка (Buzz) должен проигрываться даже если вкладка не активна.

### 2. Виртуальные Окна (Virtual OS Experience)
- **Цель**: Превратить интерфейс из «вкладки браузера» в «рабочий стол».
- **Механика**: Каждый чат — это отдельное перетаскиваемое и изменяемое в размерах окно внутри браузера (аналог десктопной ОС).
- **Библиотеки**: Исследование `react-draggable` и `react-resizable` для реализации оконного менеджера.

### 3. Автономный Десктоп-Клиент (Electron Build)
- **Цель**: Выпуск портативных сборок (`.exe`, `.dmg`, `.AppImage`). Как в инстллируемой так и в портабельной версии для хранения на флешке.
- **Преимущества**: Поддержка системного трея, иконок в панели задач и работы с нативными API операционной системы.

### 4. Групповые Чаты и Каналы (Advanced Groups)
- **Цель**: Реализация полноценных комнат по интересам.
- **Шифрование**: У каждого канала будет свой секретный ключ, который менеджеры канала (админы) будут раздавать новым участникам через зашифрованные конверты (по аналогии с текущей реализацией Общего канала).

### 5. Файловый Обмен (P2P File Transfer)
- **Цель**: Пересылка изображений и файлов напрямую между пользователями через WebRTC (DataChannels).
- **Security**: Файлы не проходят через сервер и не сохраняются в логах.

## Исследования (Research)
- **Post-Quantum Crypto**: Мониторинг новых стандартов NIST для обновления Kyber на более совершенные версии KEM.
- **Storage**: Локальное хранение истории переписки (IndexDB) с обязательным шифрованием всей базы данных мастер-паролем пользователя.

## Виджет в нижней части основного окна
- **Цель**: Виджет из магазина виджетов, который будет отображаться в нижней части основного окна. Как на скриншоте _Docs\NEW\GUI-samples\GUI-RU.png

## Настраиваемые стили и моды
- **Цель**:Сделать возможность скачать пакет с модом и стилями и установить одной кнопкой или переключаться между уже установленными модами. Чтобы можно было менять внешний вид мессенджера на ICQ 98, ICQ 2003, ICQ 2005 и т.д. Или кастомные моды пользователей из магазина модов. Моды должны быть безопасными и не должны содержать вредоносный код, иметь сертификацию от разработчика и распространяться через официальный магазин - монетизация. Моды должны быть совместимы с разными версиями мессенджера. 

## UIN  внешний вид
- **Цель**: UIN как и в ICQ должен разбиваться дефисами на группы из 3 знаков. Например 123-456-789. 

## Поддержка 2-3 видеокамер при звонке
- **Цель**: Реализовать возможность подключения 2-3 видеокамер при видеозвонке. Получатели смогут выбирать, какую камеру смотреть или все сразу - использовать одну камеру в качестве основного онка (говорящий респондент) и вторую/третью в качестве дополнительных (например, общий вид переговорной комнаты и стола, крупное изображение рабочего места, документа, демонстрируемого изделия....). Принимающий пользователь сам может выбирать, какую камеру смотреть в качестве основной, а какие показывать на маленьких экранах (как в скайпе в конференции: говорящий - крупно, все остальные - маленькими изображениями). 

## Транскрибация аудио и видеозвонка
- **Цель**: При аудио- и видеозвонках вести транскрибацию в реальном времени и сохранять в истории чата. Например, стандартное окно с двумя полями - чат и поле ввода как в обычном чате, превращается в окно с видеоизображением вверху и потоком транскрибации внизу. На этапе завоевания рынка - бесплатно, затем по подписке. Использовать штатные средства windows для транскрибации. Использовать API от яндекса или гугла. по подписке - возможность подключения пользовательских или предоставляемых нами моделей транскрибации. 

## Чёрный ящик - цепочка прокси серверов
- **Цель** Функция сборки цепочки из прокси серверов между пользователем и сигнальным сервером, а затем между пользователями для p2p соединения. Сервера выбираются из списка пользователем в открытой части мессенджера, но сама настройка соединения происходит в чёрном ящике. 

## Фидонет - иерархия
- **Цель** участники фила могут писать свободно в конфу фила, сторонние участники в конфу фила могут писать только после модерации  аристом фила.Арист фила может писать в конференции всех фил своего демоса и конференцию аристов своего демоса. 
Пример:
V:KKKK/DDDD.PPPP-MMMM   
Ветор:Курия/Демос.Фила-Монас

При этом любой неофит или анонимный пользователь можен читать все конференции. Возможно, надо будет сделать плебейские конференции, открытые для всех без ограничений

## Фидонет - адресация
- **Цель** Номер ФИДО это не UIN в полном смысле этого слова. Это скорее, второе имя. Приём сообщений всё равно осуществляется по основному UIN. Владелец фидо-номера сам определяет - показывать ли в профиле свой фидо-номер или нет. Если показывает, то другие пользователи могут отправлять ему сообщения на фидо-номер. В конференциях все сообщения подписываются только фидо-номером. 

## Скрытые чаты, фальшивые чаты
- **Цель** Скрытые чаты - это чаты, которые не отображаются в списке чатов и доступны только по паролю. Фальшивые чаты - это чаты, которые отображаются в списке чатов при вводе заранее заданного неправильного пароля. 
- **Механика**: в настройках, в разделе безопасность задаётся два пароля - истинный и ложный. При вводе истинного пароля открываются все чаты. При вводе ложного пароля открываются только фальшивые чаты. Можно сделать несколько уровней вложенности или несколько групп паролей, например при одном - открываются только чаты с друзьями о рыбалке, при другом - чаты с любовницами, при третьем - все чаты. Паролем можно закрыть как отдельные чаты, так и всего контакта со всеми его чатами.
