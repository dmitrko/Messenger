# Архитектура Сервера (Node.js + WebSocket)

## Обзор
Сервер NostaChat (`@nostachat/server`) — это легковесный координатор на базе **Express** и библиотеки **ws**. Он спроектирован так, чтобы не иметь собственной базы данных, что минимизирует риски утечки данных. Все состояние хранится в оперативной памяти и очищается при перезапуске или отключении клиентов.

## Функционал

### 1. Сигналинг (WebRTC Signaling)
Сервер выполняет роль ретранслятора сигнальных пакетов (`offer`, `answer`, `candidate`). 
- **Privacy**: Сервер не анализирует содержимое сигнальных пакетов, он лишь пересылает их по целевому UIN.

### 2. Распределение Ключей Комнаты (Room Key Distribution)
Это ключевой элемент для работы зашифрованного общего чата:
1. При старте сервер создает случайный `PUBLIC_ROOM_KEY` (32 байта).
2. При регистрации (`register`) сервер использует открытый ключ пользователя (`publicKey`) для создания анонимного зашифрованного конверта (`sodium.crypto_box_seal`).
3. Запечатанный ключ отправляется пользователю в ответе `registered`.
4. Таким образом, даже если злоумышленник перехватит трафик при регистрации, он не сможет получить ключ комнаты.

### 3. Реле-режим (Relay Mode)
Если прямое P2P-соединение между пользователями невозможно (например, из-за NAT), сервер пересылает зашифрованные пакеты (`direct_message`).
- **Logic**: При получении `direct_message`, сервер проверяет наличие флага `isEncrypted`. Если он установлен, сервер пишет в лог `[Encrypted E2EE Content]` и пересылает данные без изменений.

## Информационная безопасность
Согласно философии проекта, сервер **никогда**:
- Не логирует текст личных сообщений.
- Не хранит IP-адреса пользователей после отключения.
- Не имеет доступа к секретным ключам пользователей.

- Реализован проброс флага `isEncrypted` при рассылке сообщений в общем канале для корректной дешифровки у получателей.
