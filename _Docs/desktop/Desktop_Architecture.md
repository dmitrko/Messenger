# Архитектура Desktop-клиента (React + Vite)

## Обзор
Клиентская часть NostaChat реализована как современное одностраничное приложение (SPA) на базе **React** и **Vite**. Интерфейс стилизован под классический ICQ, но под капотом он использует современные браузерные API и криптографические библиотеки.

## Ключевые компоненты

### 1. Управление состоянием и Ref-ы
Для обеспечения бесперебойной работы WebSocket и WebRTC в условиях частого обновления UI, мы используем комбинированный подход:
- **useState**: Для данных, требующих мгновенного обновления интерфейса (список сообщений, список пользователей).
- **useRef**: Для критических данных в асинхронных колбэках. Это позволяет избежать проблемы **stale closures** (когда обработчик сообщения видит старую версию состояния, например, пустые ключи).

### 2. P2P-соединение (SimplePeer)
Реализовано через оболочку `WebRTCService`. Процесс установки соединения:
1. Клиент посылает `offer` через сервер-сигналинг.
2. Получатель отвечает `answer`.
3. После установки прямого дата-канала, сообщения передаются минуя сервер.
4. **Индикация**: В заголовке чата отображается `P2P` (прямое соединение) или `Relay` (через сервер).

### 3. Гибридное шифрование (E2EE)
Клиент использует `libsodium` для классического X25519 и `crystals-kyber-js` для постквантовой защиты.
- **Registration**: Генерируются две пары ключей (Kyber + Classic). Публичные ключи отправляются на сервер.
- **Message Send**: Сообщение шифруется комбинированным ключом.
- **Encryption Flow**: Используется `hybridEncrypt(message, recipientPK, senderSK)`.

### 4. Общий канал (Group Encryption)
Для общего чата используется симметричный ключ (SecretBox), который доставляется каждому пользователю в зашифрованном анонимном «конверте» (`crypto_box_seal`) при регистрации.

## Важные технические решения
- Решена ошибка «invalid ct size» путем исправления порядка байтов при инкапсуляции Kyber (`[ct, ss]` вместо `[ss, ct]`).
- Реализована синхронная инициализация `roomKeyRef` в обработчике регистрации для предотвращения потери первых сообщений.
